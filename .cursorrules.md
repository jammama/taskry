# Taskry 프로젝트 마스터 규칙 파일 (PM 지침)

# 1. 프로젝트 정체성 및 목표 (Context & Philosophy)
- 프로젝트명은 **Taskry**이며, 개발 중인 앱은 **게이미피케이션 기반 To-Do 웹앱**입니다.
- 핵심 철학은 **'Personal Flow Engine'**이며, 모든 UX/UI는 **Focus, Rhythm, Catalyst** 세 가지 스탯을 중심으로 설계되어야 합니다.
- **MVP 목표:** Planning Mode (할 일 관리)와 Micro Reward 시스템의 완벽한 구현입니다.
- **항상** 현재 작업이 프로젝트의 **최종 디자인 레퍼런스(다크 모드 + 네온)**와 일치하는지 확인하십시오.

# 2. 기술 및 언어 제약 조건 (Technology & Language)
- **프레임워크:** **SvelteKit** 문법과 구조를 사용해야 합니다.
- **언어:** **TypeScript(.ts) 사용은 엄격히 금지**됩니다. 모든 코드는 **JavaScript(.js)**로 작성하십시오.
- **오프라인 우선 (PWA):** 모든 핵심 To-Do 로직(입력, 완료, 읽기)은 PWA의 **IndexedDB**를 통한 오프라인 동작을 **필수**로 지원해야 합니다. 메모리 저장소는 임시로만 사용 가능하며, IndexedDB를 최우선 데이터 소스로 간주하십시오.
- **백엔드:** Supabase 연동을 가정합니다. (현재 단계에서는 로컬 스토리지에 집중)

# 3. 디자인 및 스타일 가이드 (Design & Styling)
- **테마:** **Dark Mode + Neon Glow** 테마를 엄격히 준수하십시오. 일반적인 라이트 모드 또는 부트스트랩 스타일은 사용하지 마십시오.
- **CSS 변수 사용:** 모든 색상은 `src/app.css`에 정의된 **CSS 변수**를 통해 참조해야 합니다. 절대 하드코딩된 색상 코드(예: `#FFFFFF`, `black`)를 사용하지 마십시오.
- **UX 철학 (Juicy UI):** 할 일 완료(체크) 시, **Svelte Transition**과 **CSS Keyframes**를 활용하여 과장되고 시각적으로 만족스러운 애니메이션 효과(Scale, Bounce, Glow)를 구현하십시오.
- **레이아웃:** `src/routes/+page.svelte`는 **'Planning Mode'**의 **단일 모바일 앱 레이아웃**만 포함해야 합니다. 'Reward Mode' 관련 코드는 별도의 라우트나 컴포넌트에서만 다루십시오.

# 4. 파일 및 경로 지정 규칙 (File Structure)
- **Store 경로:** 데이터 저장소 파일은 반드시 `$lib/stores/todoStore.js` 경로를 사용하십시오.
- **유틸 경로:** Task 분류 함수는 반드시 `$lib/utils/taskClassifier.js` 경로를 사용하십시오.
- **컴포넌트 경로:** 모든 재사용 가능한 UI 요소는 `$lib/components/` 내에 배치하십시오.

# 5. 작업 워크플로우 (Git & Commit)
- 모든 커밋 메시지는 **해당 이슈 번호**를 참조하고, 변경 사항을 명확하게 설명해야 합니다.
- 커밋은 최대한 **작고 원자적(Atomic)**으로 유지하여 코드 리뷰를 용이하게 하십시오. (하나의 커밋은 하나의 기능 구현에 집중)
