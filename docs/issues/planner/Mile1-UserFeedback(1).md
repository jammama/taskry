# Milestone 1 사용자 피드백

**작성일**: 2024년  
**마일스톤**: Milestone 1 - Planning Mode 기본 구현  
**상태**: 📋 검토 중

---

## 📋 피드백 개요

현재까지 구현된 Planning Mode 기능을 사용하면서 발견된 불편한 점과 개선이 필요한 사항들을 정리한 문서입니다.  
이 피드백을 기준으로 신규 추가/수정 작업을 작성합니다.

---

## 🔧 변경 사항 (UI/UX 개선)

### 1. 수정 기능 UI 개선
**현재 상태**: 수정 모드 진입 시 저장/취소 버튼이 하단에 표시됨

**피드백 내용**:
- 확인(저장) 버튼은 불필요함 (Enter 키로 대체 가능)
- 취소 버튼은 인풋창 오른쪽 상단에 배치 (앱 알림 표시 아이콘처럼)

**변경 사항**:
- [ ] 저장 버튼 제거 (Enter 키로 저장 유지)
- [ ] 취소 버튼을 인풋창 오른쪽 상단으로 이동
- [ ] 취소 버튼 아이콘 스타일 조정 (X 또는 닫기 아이콘)

**관련 파일**:
- `src/lib/components/PlanningMode.svelte` (편집 모드 UI)

---

### 2. 삭제 애니메이션 조정
**현재 상태**: 삭제 시 왼쪽으로 -40px 정도 밀림, clip-path로 왼쪽 부분 자름

**피드백 내용**:
- -40px은 과도함
- -10px 정도면 가시성에 충분
- clip-path 제거 고려 (약간 밀리는 정도면 미관 해치지 않음)

**변경 사항**:
- [ ] 삭제 시 밀림 정도를 -40px → -10px로 조정
- [ ] clip-path 제거 검토
- [ ] 삭제 애니메이션 부드러움 개선

**관련 파일**:
- `src/lib/components/PlanningMode.svelte` (삭제 애니메이션)

---

### 3. Task 제목 글자수 제한 및 레이아웃 보호
**현재 상태**: 제목 길이 제한 없음, 긴 제목 시 레이아웃 벗어날 수 있음

**피드백 내용**:
- Task는 간결함이 생명
- 입력/수정 글자수 제한 필요
- 긴 제목도 레이아웃을 벗어나지 않도록 처리

**변경 사항**:
- [ ] 제목 최대 글자수 제한 정의 (예: 50자 또는 100자)
- [ ] 입력 시 글자수 제한 적용
- [ ] 긴 제목 처리 방식 결정 (말줄임표, 자동 줄바꿈, 폰트 크기 조정 등)
- [ ] 레이아웃 벗어남 방지 (CSS `text-overflow`, `word-break` 등)

**관련 파일**:
- `src/lib/components/PlanningMode.svelte` (입력/수정 필드)
- `src/lib/components/NewTaskInput.svelte` (새 할 일 입력)
- `src/lib/stores/todoStore.js` (유효성 검사)

**고려사항**:
- 글자수 제한 값은 사용자 테스트를 통해 최적값 결정
- 한글/영문/숫자 등 문자 종류별 처리 방식 고려

---

### 4. 전반적인 애니메이션 효과 강화
**현재 상태**: 애니메이션 효과가 부족하여 보상이 즐겁지 않음

**피드백 내용**:
- 파티클 효과 부족
- 체크박스 누르는 애니메이션 부족
- 보상 노출 애니메이션 부족
- 전반적으로 '보상'이 즐겁다는 느낌이 들지 않음

**변경 사항**:
- [ ] 체크박스 클릭 애니메이션 강화 (Bounce, Scale, Glow 효과)
- [ ] 파티클 효과 추가 (폭죽, 별, 코인 등)
- [ ] Micro Reward 팝업 애니메이션 강화
- [ ] 완료 시 화면 살짝 흔들림 효과 (선택)
- [ ] Flow Energy 표시 애니메이션 개선
- [ ] 전반적인 애니메이션 타이밍 및 강도 조정

**관련 파일**:
- `src/lib/components/PlanningMode.svelte` (체크박스, 보상 팝업)
- `src/lib/components/RewardPopup.svelte` (보상 팝업)

**참고 문서**:
- `docs/Milestone-Basic(Detail).md` - Milestone 1 Phase 1.2: Juicy UI 애니메이션 구현
- `docs/uiuxGuide.md` - Juicy UI 가이드라인

## ➕ 추가 사항 (신규 기능)

### 5. 완료된 할 일 분리 표시
**현재 상태**: 진행할 할 일과 완료된 할 일이 한 리스트에 함께 표시됨

**피드백 내용**:
- 완료된 할 일을 별도 리스트로 하단에 표시
- 접고 펴기 기능 필요

**추가 사항**:
- [ ] 완료된 할 일을 별도 섹션으로 분리
- [ ] 완료된 할 일 섹션 접기/펴기 기능
- [ ] 접기/펴기 상태를 IndexedDB에 저장 (사용자 선호도 기억)
- [ ] 완료된 할 일 섹션 UI 디자인 (접힘/펼침 상태)

**관련 파일**:
- `src/lib/components/PlanningMode.svelte` (리스트 분리 로직)
- `src/lib/stores/todoStore.js` (필터링 로직)

**고려사항**:
- 완료된 할 일 표시 방식 (항상 표시 vs 접기/펴기)
- 완료된 할 일 자동 정리 정책 (예: 7일 후 자동 삭제 또는 아카이브)

---

### 6. Task 순서 변경 (드래그앤드롭)
**현재 상태**: Task 순서 변경 불가능

**피드백 내용**:
- 우선순위와 중요도는 사용자가 정할 수 있어야 함
- 드래그앤드롭이 가장 현명한 방법

**추가 사항**:
- [ ] 드래그앤드롭 라이브러리 선택 (예: dnd-kit, svelte-dnd-action)
- [ ] Task 순서 변경 기능 구현
- [ ] 순서 정보를 IndexedDB에 저장 (`order` 또는 `position` 필드)
- [ ] 드래그 중 시각적 피드백 (그림자, 배경색 변경 등)
- [ ] 드롭 시 애니메이션 효과

**관련 파일**:
- `src/lib/components/PlanningMode.svelte` (드래그앤드롭 로직)
- `src/lib/stores/todoStore.js` (순서 업데이트 함수)
- `src/lib/stores/todoStore.js` (데이터 구조에 `order` 필드 추가)

**고려사항**:
- 모바일 환경에서의 드래그앤드롭 UX (터치 제스처)
- 성능 최적화 (많은 Task가 있을 때)
- 순서 변경 시 애니메이션 부드러움

## 💭 고려 사항 (향후 기능)

### 7. 날짜 기능 UI 설계
**현재 상태**: 날짜 기능은 Milestone 2에서 추가 예정

**피드백 내용**:
- 날짜 기능 추가 시 UI 복잡도 우려
- 현재 Task UI가 깔끔한데, 날짜 추가로 복잡해질까 걱정
- 어떤 방식으로 정보를 보여줄 수 있을지 고려 필요

**고려 사항**:
- [ ] 날짜 표시 방식 설계 (아이콘, 텍스트, 색상 등)
- [ ] Task UI 간결성 유지 방안
- [ ] 날짜 정보 표시 위치 (제목 옆, 하단, 아이콘 등)
- [ ] 날짜가 없는 Task와 있는 Task의 시각적 구분
- [ ] 날짜 필터링 UI 설계

**관련 문서**:
- `docs/Milestone-Basic(Detail).md` - Milestone 2: ToDo 기능 고도화

**디자인 제안**:
- 날짜 아이콘을 작게 표시 (예: 📅 아이콘 + "오늘", "내일", "12/25" 등)
- 날짜가 있는 Task만 작은 배지 형태로 표시
- 날짜 정보는 호버 시 상세 표시
- 색상으로 날짜 구분 (오늘/내일/과거/미래)
