## 📖 기능 개요

### 목적
할 일이 완료되어도 숫자(order)가 변경되지 않도록 고정합니다. order 변경은 '할 일 안에서 순서를 변경했을 경우'로만 한정하여 사용자 로직에 맞게 동작하도록 합니다.

### 사용자 스토리
- **As a** 사용자
- **I want** 할 일이 완료되어도 숫자가 변경되지 않는 것
- **So that** 할 일의 순서가 일관되게 유지되고 사용 로직에 맞게 동작한다

---

## 🎯 기능 요구사항

### 핵심 기능
1. 할 일이 완료되어도 order 값이 변경되지 않음
2. order 변경은 '할 일 안에서 순서를 변경했을 경우'로만 한정
3. 완료된 할 일의 숫자 표시는 원래 order를 기준으로 함

### 상세 요구사항
- **order 고정**: `completeTodo` 함수에서 order 값을 변경하지 않음
- **숫자 표시**: 완료된 할 일의 숫자는 `activeTodos.length + index + 1`이 아닌 원래 order를 기준으로 표시
- **순서 변경 제한**: order 변경은 드래그앤드롭으로 할 일 순서를 변경했을 때만 발생
- **완료/미완료 전환**: 할 일이 완료되거나 완료 해제되어도 order는 그대로 유지
